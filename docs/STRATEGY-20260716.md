## 1. Performance improvements

**Where we stand (measured, spec 08 / PRODUCT.md):** cold start 0.94 s, live
caption p95 0.53 s, post-meeting summary 3.8 s, detail first-content 91.87 ms
at 5k segments, exact search p95 30.99 ms at 100k segments, drift 4 ms/22 min,
RAM 569–795 MB peak while recording → 140–160 MB idle. These already beat every
cloud competitor's UX-visible latency; the work left is protecting those
numbers and removing the last known cliffs.

**Known cliffs:**

- **RAG embeddings are brute-force O(n)** (GAPS T5). Fine at 1k meetings;
  `ask` degrades past that. The FTS side was already fixed (D81 bounded
  candidates + reciprocal-rank fusion).
- **Embedded MLX summary spikes ~2.4 GB transiently** (released after 120 s).
  Acceptable, but the spike is the single worst memory moment in the app.
- **Spotlight reindex is full, not incremental** (deliberately deferred to
  Band 4's outbox work).
- **No CI perf regression gate**: every number above is reproducible via CLI
  benches, but nothing fails automatically when a slice regresses one.

**Actions**

- [ ] **P1/M — Perf regression gates in CI.** A `bench-gate` job that runs
  `--bench-startup`, `bench-scale`, and the detail first-content bench against
  budget thresholds (cold < 1.5 s, search p95 < 50 ms, detail < 300 ms) and
  fails the PR on regression. Needs the self-hosted runner conversation
  (GAPS T7) or a nightly scheduled job on the maintainer's Mac.
- [ ] **P1/M — sqlite-vec (or HNSW-in-SQLite) for embeddings**, gated on a
  measured failure: extend `bench-scale` to time semantic `ask` at 1k/5k/10k
  meetings first; adopt the index only if p95 breaks its budget (avoid
  speculative complexity — the D81 lesson).
- [ ] **P2/M — Band 4 outbox indexing** (already planned): incremental
  Spotlight + embedding updates driven by row-change events instead of full
  reindex on `libraryVersion` bumps.
- [ ] **P2/S — MLX residency knob.** Expose the 120 s idle-release as an
  advanced setting (30 s / 120 s / until quit) for 8 GB Macs; measure summary
  cold-start delta per option in the bench.
- [ ] **P2/S — Model prewarm on calendar intent**: when a calendar meeting
  starts in < 10 min and the app is open, prewarm Parakeet (the AEC warm-up
  pattern already exists in `MicrophoneSource.warmUp`). Perf work that doubles
  as the "prepared before you join" signature moment.

## 2. Architecture and patterns that serve performance

The Band 0–3 refactor already picked the right patterns — the proof is
measured: scoped GRDB observations cut Meeting Detail first-content from
522 ms to 92 ms and `MeetingHealth` p95 from 348 ms to 10 ms. The guidance is
to **finish the program, not redesign it**.

**Patterns already in place to extend, not replace:**

| Pattern | Where it lives | Extend it to |
|---|---|---|
| Scoped observation per feature (no global `libraryVersion` invalidation) | Library/Insights/Detail models | Settings panes, menu-bar panel |
| Use-cases in `ApplicationKit` (Start/Stop/Refine/Import/Regenerate) | Band 1–2 slices | Export, enrollment, dictation session |
| Single-flight priority scheduler for ML (D29) | `IntelligenceScheduler` | Batch Companion refresh + chapter titling (currently serial FM calls) |
| Durable jobs + provenance (`ProcessingJob`, `GenerationRun`) | Band 1/3 | Any new long-running work — never a bare `Task` |
| Egress gateway with classified, consented events | Band 3 slices 3F–3G | Every future network feature, no exceptions |
| Capability types (`FoundationModelsCapability`) | Sequoia unit 4 | Translation framework, SpeechAnalyzer, per-OS gates |
| Role-scoped model loading (`loadTranscriberIfNeeded` / `loadDiarizerIfNeeded`) | Sequoia unit 5 | Whisper variants; a future unified residency ledger |

**One new proposal — `ModelResidencyLedger` (P2/M):** today each engine tracks
its own idle-generation counters. A single owner that knows every resident
model, its bytes, its last use, and applies one LRU policy would (a) simplify
the per-engine logic, (b) let the UI honestly show "what is using RAM right
now" (pairs with the privacy receipt for a "resources receipt"), and (c) give
8 GB Macs a global budget instead of per-engine heuristics.

**Actions**

- [ ] **P1/S — Finish Band 3H (privacy receipt)** — architectural keystone that
  is also the sales artifact (§15).
- [ ] **P2/M — `ModelResidencyLedger`** as above, measured before/after.
- [ ] **P2/S — Batch FM micro-calls** (chapter titles, Companion refresh) under
  one scheduler entry with cancellation, instead of N serial `.background` runs.

## 3. Compatibility across versions

**Today:** macOS 14.4+ (the `CATap` floor — honest and correct), Apple Silicon
only, Foundation Models features require macOS 26. The Sequoia stabilization
interrupt (units 1–6) just made the 14.4–15.x experience first-class: feasible
summary defaults, no silent engine fallthrough, typed setup routes, and
Companion honestly absent instead of an impossible toggle.

**Recommendations:**

- **Do not chase Intel.** ANE/MLX are the product. Document it as an anti-goal
  (§10) — the appcast already declares `arm64`.
- **Do not lower the 14.4 floor.** The tap IS the app.
- **Treat "Sequoia-class Mac" as the largest real segment** for the next two
  years. Every new intelligence feature ships with its capability enum and a
  local fallback (MLX/Ollama) or an honest absence — the unit-4 pattern.
- **The biggest compatibility gap is Companion-on-Sequoia**: it requires FM
  today. An MLX-backed classifier/answerer path would light up the flagship
  feature for every Apple Silicon Mac. (Also a differentiation move — see §15.)
- **Locales:** EN/ES complete with a per-string test gate. Portuguese (pt-BR)
  is the cheapest high-value add (LATAM adjacency; catalog infra exists).

**Actions**

- [ ] **P1/L — Companion on Sequoia via MLX** (classifier + knowledge answers
  through `MLXSummaryProvider`-class runtime; context answers already run on
  local RAG). Gate: same < 5 s budget on an M1.
- [ ] **P1/S — Compatibility matrix in README/site**: one table — feature ×
  macOS 14.4/15/26 — generated from the capability types so it cannot rot.
- [ ] **P2/M — pt-BR localization** after the next catalog freeze.

## 8. Pricing model — assessment and recommendation

**Current target (PRODUCT.md): one-time ~$69, launch $49, generous FREE.**

Assessment against the July 2026 market (see §9): cloud subscriptions cluster
at **$8–20/u/mo annual, median ≈ $15** ($16–30 monthly); lifetime/one-time
survives — and thrives — only among indie native Mac apps (MacWhisper €59
lifetime is the proven benchmark; superwhisper $249 lifetime; Whisper Notes
$6.99); Granola now monetizes a 25-note free-history cap and Fathom clawed
back its free tier, feeding exactly the subscription-fatigue the one-time
model converts. Notably, **"free MIT core + one-time paid Pro" is untried in
the entire niche** — Meetily and Anarlog both chose open-core with $10–15/u/mo
subscriptions. Being the first pay-once open-source meeting assistant is
itself a marketing hook. **Verdict: the model is right. The number ($69, $49
launch) is right — comfortably above MacWhisper (more product) and far under
superwhisper. The gaps are policy details:**

- [ ] **P0/S — Write the upgrade policy before launch**: v1.x updates free
  with the license; a future v2 is a paid upgrade at ~50% for owners
  (Things/Sketch pattern). Announcing it up front converts subscription-fatigue
  buyers who fear abandonment.
- [ ] **P0/S — Keep FREE genuinely complete** (unlimited local record/
  transcribe/diarize/summarize, as PRODUCT.md already commits). The FREE tier
  is the top of the funnel and the moral high ground; never move core capture
  behind pay.
- [ ] **P1/S — Team pack** (5 licenses, ~$199) — zero infra, pure SKU.
- [ ] **P1/S — EDU/OSS-maintainer discount** (30–40%) — cheap goodwill in the
  exact communities that amplify OSS tools.
- [ ] **Do not join Setapp** (subscription framing contradicts the pitch,
  ~30% cut, and the buyer overlap with Homebrew users is small).
- [ ] **P2/S — Optional paid "supporter" tier on GitHub Sponsors** for the
  FREE users who want to pay without needing PRO.

## 9. Competitive landscape (web-checked Jul 16, 2026) and how to win

> Live-web research pass (all sources accessed Jul 16, 2026; GitHub stars via
> the API the same day). PRODUCT.md keeps the deeper per-competitor teardown;
> this section carries what changed and what it means.

**The market in one paragraph.** The category consolidated upward: Granola is
at a $1.5B valuation (Series C $125M, Mar 2026), Fireflies $1B and profitable,
Otter past $100M ARR and pivoting to enterprise "knowledge engine", Plaud at
$250M ARR on hardware. Market ≈ $3.9–4.3B in 2026 growing ~25%/yr; enterprise
is ~70% of spend but **prosumer is the fastest-growing segment — and the
well-funded players are all fleeing it upmarket**. Bot-free capture became
table stakes (Granola, Krisp, Jamie, Circleback, Bluedot, Notion followed);
**fully-local processing did not** — that is the active frontier and exactly
Portavoz's wedge (Granola's own security page confirms Deepgram/AssemblyAI +
OpenAI/Anthropic in its path).

**Three tailwinds that appeared in 2025–26 (use them in copy):**

1. **The bots are being evicted**: Microsoft rolls out Teams bot detection /
   lobby quarantine May→Q3 2026, hitting Otter/Fireflies/Fathom/Read.ai
   tenants. Bot-based competitors lose their capture path; device-side capture
   is structurally immune.
2. **Cloud notetakers are in legal trouble**: Otter faces a federal class
   action over training on recordings (NPR, Aug 2025); the NYC Bar's Formal
   Opinion 2025-6 (Dec 2025) warned lawyers off cloud notetakers;
   universities are blocking bots. Every such event manufactures demand for
   "audio never leaves the machine".
3. **Free tiers are shrinking** (Granola capped history at 25 notes; Fathom
   capped advanced summaries at 5/mo) — subscription fatigue plus clawbacks
   make "pay once, own it forever" land harder than ever.

**Condensed field (Jul 2026):**

**Gaps nobody fills (verified Jul 2026) — Portavoz maps onto nearly all:**

1. 100% local + open source + **polished native macOS** app (Meetily = rough
   Tauri; Anarlog = brand-churned, cloud-leaning).
2. **Open-source live in-meeting companion** — Amurex is dormant (last push
   May 2025); Cluely is radioactive; Otter's agent is cloud enterprise. The
   slot is empty; Portavoz's Companion is the only live entrant.
3. **Bilingual EN/ES as product** (not "100 languages" ASR marketing) — the
   LATAM/Spain professional market is structurally unserved.
4. **One-time purchase for a full meeting assistant** (MacWhisper proves the
   price point for transcription alone).
5. **On-device cross-meeting RAG/Ask** — the enterprise clouds do it
   server-side; nobody does it locally. Portavoz already ships it.
6. **Post-Rewind orphans**: Meta bought Limitless (Dec 2025) and shut Rewind
   down — privacy-minded local-memory users lost their product and nobody has
   courted them.
7. **Regulated-prosumer compliance story** (lawyers/doctors/finance) — the
   NYC Bar opinion created demand only EU-cloud Jamie half-answers.

**Threats to respect:** Apple absorbing the low end (macOS 26 already records
calls on-device; system-audio capture could come in macOS 27 — keep living
above the floor per PRODUCT.md's rule); Granola's war chest and defensive
"local-first vs cloud" content (it feels the attack); Meetily's star-count
gravity in the OSS lane; SEO in this niche is a content war among
micro-competitors.

**How to win (positioning, unchanged and validated):** nobody combines
local-first + speaker identity + developer workflow + bilingual + one-time +
MIT. Every competitor holds ~2 of the 6. The strategy is not to out-feature
Otter; it is to be the **obvious** choice for the intersection: privacy-
conscious Mac professionals, developers, and bilingual teams — then expand
outward from a defensible core.

- The "local open source" throne is **vacant** (Anarlog pivoted away; Meetily
  is cross-platform Tauri with zero RAG/chat; MacParakeet is dictation-first
  GPL). Portavoz can own "the serious native one".
- The Companion window is real and closing: Teams Facilitator lands
  ~Aug–Sep 2026. Shipping the field-verified, honest, local Companion story
  BEFORE that (with latency receipts) frames Microsoft as validating us.
- Apple's floor (Notes transcription, SpeechAnalyzer) rises yearly: the rule
  in PRODUCT.md holds — never build a core feature Apple will obviously
  absorb; keep living above the floor (identity, history, workflow, bilingual).

## 10. What NOT to become (anti-goals)

1. **No bots that join calls** — the category's most-hated pattern.
2. **No accounts, no cloud-default, no telemetry-default** — ever.
3. **No subscription for local features.** If recurring revenue is ever
   needed, it can only price genuinely recurring costs (a hosted relay), never
   local capability.
4. **No archive hostage** (the Granola lever). FREE users' data is never
   time-locked, degraded, or export-restricted.
5. **No rug-pull relicensing.** MIT stays MIT (the Anarlog pivot vacated the
   throne we now claim; repeating it would destroy the moat).
6. **No Electron/Tauri rewrite, no Windows port** while the Mac edge is the
   product. Cross-platform dilution is how the natives die.
7. **No Cluely-style covert overlay.** The Companion is transparent,
   disclosed, and never speaks for the user. "Cheating tool" stigma is a
   brand-killer in the enterprise-adjacent market.
8. **No enterprise checkbox sprawl** (SSO/DLP/compliance dashboards) before
   the prosumer core is won.
9. **No feature Apple ships "basic" within ~2 years** as a headline.
10. **No dark-pattern growth** (nag screens, forced sharing, SEO spam).

## 11. Marketing strategies (concrete, evidence-backed)

## 12. Language / stack changes for differentiation?

**No change. The stack IS the differentiation.** Swift 6 strict + SPM +
CoreML/ANE + MLX is what produces 0.53 s live captions at 600 MB RAM while the
Tauri/Electron competitors ship heavier and slower; PRODUCT.md's measured
table is the proof. Recommendations at the edges only:

- Adopt the **Xcode app target** (M14a) for AppIntents/extensions — SPM
  remains the source of truth for Kits.
- Keep **CLI + MCP** as first-class Swift targets (developer moat).
- Consider **Rust only if** a future cross-platform *core* (not UI) must be
  shared with iOS+server — currently nothing qualifies.
- The honest differentiator to *say out loud*: "native Swift, no web view,
  no sidecar processes — and here are the numbers".

## 14. Is the architecture maintainable and fitting? How to improve it

**Yes — unusually so for a solo project**: strict Swift 6, 650+ tests
including characterization suites, an 80+ entry decision log, as-built specs
separated from plans, a refactor program with a feature-parity contract, and
provenance/egress architecture most teams never build. The risks are
elsewhere:

- **Bus factor = 1.** Mitigations: CONTRIBUTING.md + labeled good-first
  issues; keep AGENTS.md the single operating guide (it already is); the
  decision log is the onboarding asset.
- **CI cannot run model-gated tests** (GAPS T7) — a monthly manual full-gate
  run on the maintainer's Mac, written into RELEASING.md, is the cheap fix.
- **UI-test harness fragility** (testmanagerd wear causes false mass
  failures): add `killall testmanagerd` to `test-ui-preflight` and a
  one-retry policy for the known "row not hittable" signature. Document the
  tree-stability fingerprint check used during concurrent-agent QA.
- **Docs sprawl**: refactor-20260714.md is load-bearing but huge; once Band 4
  lands, fold the completed bands into ARCHITECTURE.md and keep only live
  bands in the plan doc.

## 15. WOW-factor ideas (novel in the market)

Ranked by (uniqueness × feasibility on the existing codebase):

1. **The privacy receipt as a shareable artifact** (Band 3H + S): after each
   meeting, one glanceable card — "This meeting: 100% on-device. 0 bytes

6. **"Ask your week" as an OS-level surface** (⌘K exists): global hotkey +

## 16. Becoming world-class (the quality bar)

World-class here = **provable numbers + provable privacy + release
discipline**, sustained:

- **Publish the benchmark dashboard**: one page (site) auto-fed by the CLI
  benches per release — latency, DER, RAM, search — with the Mac model listed.
  "Measured, not promised" becomes checkable history, not a slogan.
- **Quality gates already exist — wire them to releases**: swift test +
  strict lint + UI suite + distribution verifier are all in place; RELEASING
  should require the bench suite and the field-verification checklist per
  minor release (the Jul 16 interrupt showed field feedback lands fast when
  scheduled).
- **Accessibility as a differentiator**: a VoiceOver-audited transcript view
  (live regions for captions) would be unique in the category and is the
  right thing for a tool whose output is speech; add an a11y audit milestone.
- **Security posture**: keep the honest non-sandbox statement (D78), publish
  SECURITY.md response SLOs, and add a reproducible-build story when the
  Xcode target lands; the egress-gateway + receipts architecture is already
  ahead of the market — say so publicly with the receipt artifact.
- **Support loop**: opt-in local diagnostics export (Band 3 tail) turns "it
  failed" reports into fixable evidence without telemetry.
- **The bar to hold**: every release keeps — tests green, benches within
  budget, zero Spanish-prose regressions in EN sources, localization complete,
  CHANGELOG user-facing, notarized + stapled + cask verified. That discipline,
  visible in public, IS the world-class signal for this category.
